Script-owned design

The lite version is intentionally constrained. A text-only agent supplies data and intent; the generator owns the layout, SVG, color system, easing, controls, and QA hooks.

spec.json

Small input contract: chart type, title, takeaway, source, values, unit, duration, and style.

scripts/build.py

Deterministic renderer for charts, controls, reduced motion, and frame-seek parameters.

chart_motion.html

Single-file output with no runtime dependency and browser-ready motion.

?static=1

Freezes the final frame for screenshot tests, review, and accessibility.

?t=<ms>

Seeks deterministic animation frames so motion can be inspected without guesswork.

showcase/

Publishable README page, gallery references, templates, and community QR asset.

Workflow

Four concrete steps from data to a finished animated chart page.

1

Prepare the numbers

Use labels, values, source, unit, and one sentence that names the takeaway.

2

Pick a chart

Choose from KPI, bar, column, diverging, line, multiline, area, stacked, scatter, lollipop, or dumbbell.

3

Write the spec

Follow references/spec.md. Do not hand-author HTML or chart CSS.

4

Build and review

Open the generated HTML, replay the motion, and freeze the final frame when needed.

{
  "chart": "kpi",
  "title": "ARR reached $12.4M",
  "takeaway": "ARR climbed 42% after launch.",
  "source": "Company dashboard - FY2026",

  "value_format": "0.0",
  "value": 12.4,
  "kpi_label": "annual recurring revenue",

Lite versus full

Full Data2Motion is for high-stakes custom data stories. Lite is for fast, repeatable chart motion when the agent cannot visually inspect its own output.

AreaData2Motion fullData2Motion Lite
Target modelStrong multimodal agentAny text-only agent
Chart choiceBrief-led editorial and perceptual selectionFixed chart list selected by user or agent
Motion designCustom choreography and visual QAEncoded once inside the generator
OutputFull motion package with evidenceOne JSON spec and one standalone HTML file
